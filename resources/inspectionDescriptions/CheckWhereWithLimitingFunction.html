Checks Collection.where(...).first() style method calls.

These calls after Where condition are performance impacting. The collection has to be traversed first to satisfy Where and then the limiting function will be called on the returned collection.

Instead of Use this
Where(...).first() firstWhere(...)
Where(...).single() singleWhere(...)
Where(...).whereTypeIs(...) Combine both conditions
Where(...).hasElement hasMathc(...)
Where(...).count countWhere(...)
